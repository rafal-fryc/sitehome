**2. Create scripts/generate-takeaways.ts:**
Follow the classify-provisions.ts pattern exactly. Key structure:

- **Imports:** `@anthropic-ai/sdk`, `fs` (readFileSync, writeFileSync, renameSync, readdirSync), `path`
- **SDK init:** `const anthropic = new Anthropic();`
- **Idempotency check:** `isAlreadyProcessed(data)` returns true if `data.takeaway_brief !== undefined`
- **writeJSONSafe:** Same atomic write pattern (tmp file, JSON.parse validation, rename)

- **Dry-run with custom sample selection (TAKE-05):**
  - `const DRY_RUN = process.argv.includes("--dry-run");`
  - Define `SAMPLE_IDS` array of 10 representative case file IDs (without .json extension). Select a mix covering:
    - COPPA cases (e.g., `12.25_disney`)
    - Data security (e.g., `07.19_equifax`)
    - Surveillance (e.g., `12.24_gravy_analytics`)
    - TSR/financial (e.g., `01.05_assail_inc`)
    - Edge case with empty complaint (e.g., `10.20_ntt_global_data_centers_americas`)
    - Recent vs older cases, different violation types (deceptive, unfair, both)
    - Find actual filenames by listing `public/data/ftc-files/` and selecting 10 diverse cases
  - In dry-run mode, filter to only SAMPLE_IDS files. In full mode, process all files.

- **Prompt builder function `buildTakeawayPrompt(caseData)`:**
  Extract from caseData:
  - `case_info.company.name`, `case_info.case_date.year` or parse from `date_issued`, `case_info.violation_type`, `case_info.legal_authority`
  - `complaint.factual_background` (may be empty for 1 case)
  - `complaint.representations_made[].description` (may be empty for 12 cases)
  - `complaint.counts[].title` (may be empty for 3 cases)
  - `order.provisions[].title` and `.summary` (always available)

  System prompt instructs the LLM to:
  - Generate `brief` (1 sentence, 15-25 words, violation only, no remedy, practitioner plain English)
  - Generate `full` (2-3 sentences, violation + what FTC ordered, plain English)
  - CRITICAL constraints (TAKE-04): ONLY state facts from the provided data. Do NOT invent dollar amounts, dates, or statute section numbers not in the data. Do NOT name statutes not in legal_authority. If data is sparse, write shorter/more general summary.
  - Return ONLY valid JSON: `{ "brief": "...", "full": "..." }`
  - Include 2 few-shot examples in the prompt for consistent tone:
    - Brief example: "Charged consumers without consent using dark patterns and retaliated against chargeback disputes."
    - Full example: "The company deceived consumers about how it collected and used personal data, including precise geolocation. The FTC required deletion of illegally collected data, a comprehensive privacy program, and third-party assessments for 20 years."

- **LLM call function:**
  ```typescript
  model: "claude-sonnet-4-5"
  max_tokens: 512  // Conservative for short text output
  temperature: 0   // Deterministic for consistent tone (TAKE-04)
  ```
  Parse response, strip markdown fences if present, JSON.parse the result.

- **Post-generation validation (TAKE-04):**
  After parsing LLM output, run lightweight checks:
  - Word count check on `brief`: if > 30 words, log a warning (don't block)
  - Check for `$` in output that doesn't appear in input data: log warning
  - Check for 4-digit year patterns in output not present in input: log warning
  These are warnings only, not blockers. Log as `WARN {filename}: {issue}`.

- **Apply takeaways to case data:**
  Set `data.takeaway_brief = result.brief` and `data.takeaway_full = result.full` at the top level of the case JSON (NOT inside case_info).

- **Main loop:** Same sequential pattern as classify-provisions.ts:
  - Read file, check idempotency, build prompt, call LLM, validate, apply, write (if not dry-run)
  - 300ms rate-limiting pause between API calls
  - Count classified/skipped/errors, print summary

- **Console output:** For dry-run, print the prompt (first 1500 chars) and full result for each case. For full run, print `OK {filename} -- brief: {first 60 chars}...`

Review the 10 sample outputs. If any look problematic (hallucinated details, wrong tone, too long), adjust the prompt and re-run. The dry-run MUST complete successfully before proceeding.

**Important:** Do NOT run the full batch in this task. Only dry-run. The full batch will be triggered after verifying dry-run results look good.
