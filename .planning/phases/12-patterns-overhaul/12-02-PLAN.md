```typescript
// Phase 12: Behavioral Patterns — extracted from case takeaway data

export interface BehavioralCase {
  case_id: string;
  company_name: string;
  date_issued: string;
  year: number;
  takeaway_brief: string;
  docket_number: string;
  ftc_url?: string;
  statutory_topics: string[];
  categories: string[];
}

export interface BehavioralPattern {
  id: string;
  name: string;
  description: string;
  case_count: number;
  year_range: [number, number];
  most_recent_year: number;
  most_recent_date?: string;
  enforcement_topics: string[];
  cases: BehavioralCase[];
}

export interface BehavioralPatternsFile {
  generated_at: string;
  total_patterns: number;
  total_cases_categorized: number;
  patterns: BehavioralPattern[];
}
```

**Step 2: Create extraction script**

Create `scripts/extract-behavioral-patterns.ts` with two modes:

**Mode 1: `--propose`** (first pass)
1. Read `public/data/ftc-cases.json`, extract all cases with `takeaway_brief`
2. Analyze the 285 takeaway_brief texts programmatically to identify common behavioral themes. Use keyword/phrase matching to group cases into 10-15 broad categories. The categories should capture "what the company did wrong" at a behavioral level.

   **Target categories (use these as starting points, refine based on actual data):**
   - "Deceptive Marketing Claims" — false advertising, misleading product claims
   - "Unauthorized Data Collection" — collecting data without consent or beyond stated purposes
   - "Inadequate Data Security" — failure to protect consumer data, security breaches
   - "Children's Privacy Violations" — COPPA violations, collecting from minors
   - "Unfair Billing Practices" — unauthorized charges, deceptive pricing
   - "Privacy Policy Deception" — false privacy promises, undisclosed data sharing
   - "Illegal Telemarketing" — Do Not Call violations, robocalls, TSR violations
   - "Credit Reporting Violations" — FCRA violations, inaccurate reporting
   - "Surveillance & Tracking" — undisclosed monitoring, location tracking, spyware
   - "Dark Patterns & Deceptive Design" — trick interfaces, hard-to-cancel subscriptions
   - "Unauthorized Data Sharing" — selling or sharing data without consent
   - "Identity Theft Facilitation" — enabling or failing to prevent identity theft
   - "Algorithmic Harm" — biased or deceptive AI/automated decision-making

   For each case, match against multiple categories (a case can belong to several).

   **Keyword matching approach:** For each category, define a set of keywords/phrases to match against takeaway_brief text (case-insensitive). Include both direct terms and common synonyms. A case matches a category if any keyword matches. Err on the side of inclusion — the user will review.

3. Write output to `scripts/behavioral-categories-review.json` with:
   ```json
   {
     "generated_at": "...",
     "mode": "proposed",
     "total_cases": 285,
     "categories": [
       {
         "id": "deceptive-marketing-claims",
         "name": "Deceptive Marketing Claims",
         "description": "Companies making false or misleading advertising claims about products or services",
         "keywords_used": ["misleading", "false claims", "deceptive advertising", ...],
         "case_count": N,
         "sample_cases": [
           { "case_id": "...", "company_name": "...", "takeaway_brief": "..." }
         ],
         "all_case_ids": ["case_id_1", "case_id_2", ...]
       }
     ],
     "uncategorized_cases": [
       { "case_id": "...", "company_name": "...", "takeaway_brief": "..." }
     ]
   }
   ```
   Include first 5 sample cases per category, and list all case IDs.

**Mode 2: `--finalize`** (second pass, after user review)
1. Read the (potentially user-edited) `scripts/behavioral-categories-review.json`
2. Read `public/data/ftc-cases.json` for full case metadata
3. For each approved category, build a `BehavioralPattern` object:
   - Look up each case_id in the cases data to get full metadata (date_issued, year, statutory_topics, etc.)
   - Sort cases chronologically
   - Compute year_range, most_recent_year, most_recent_date, enforcement_topics (union of all case statutory_topics)
4. Write `public/data/ftc-behavioral-patterns.json` with the BehavioralPatternsFile shape

**Script conventions:**
- Use `const fs = require("fs")` and `const path = require("path")` (same as build-patterns.ts)
- Use the `writeJSONSafe` pattern (write to .tmp, validate JSON, rename)
- Accept mode via `process.argv[2]` (either `--propose` or `--finalize`)
- Print progress to console
